ISO 27001:2022 — Control Mapping

ISO/IEC 27001:2022 specifies requirements for establishing, implementing, maintaining, and continually improving an information security management system (ISMS). The 2022 revision reorganises controls into four themes with 93 controls total. Below is the mapping of Microsoft Defender XDR capabilities to each Annex A domain.

A.5 — Organisational Controls

14 controls mapped ▼
A.5.7 Threat intelligence — Collect and analyse threat intelligence to inform security decisions Compliant XDR Threat Analytics
A.5.8 Information security in project management — Integrate security throughout project lifecycle Partial XDR Multi-Tenant Mgmt
A.5.9 Inventory of information and other associated assets — Maintain asset inventory Compliant MDE Device Inventory, MDVM Software Inventory, MDCA Shadow IT
A.5.12 Classification of information — Classify information according to sensitivity Non-Compliant DLP Integration (not enabled)
A.5.14 Information transfer — Protect information during transfer Non-Compliant MDO DMARC/DKIM/SPF
A.5.15 Access control — Ensure authorised access to information assets Compliant MDI AD Security, XDR RBAC
A.5.17 Authentication information — Manage authentication credentials securely Compliant MDI Credential Detection
A.5.23 Information security for use of cloud services — Manage cloud service security Partial MDCA App Governance
A.5.25 Assessment and decision on information security events — Triage and assess events Compliant MDE AIR, XDR Incident Queue
A.5.26 Response to information security incidents — Execute incident response procedures Compliant MDO ZAP, XDR Auto-IR
A.5.36 Compliance with policies, rules, and standards — Monitor policy adherence Partial XDR Secure Score
